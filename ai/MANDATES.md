# 🚨 CRITICAL MANDATES - NEVER VIOLATE THESE

## **PRIMARY DIRECTIVES**

### **0. QUALITY OVER SPEED - ALWAYS**
**NEVER compromise best practices for perceived time pressure:**
- ✅ Best code and solutions take priority over delivery speed
- ✅ Proper process is more important than "getting something working quickly"
- ✅ User waiting is preferable to delivering suboptimal solutions
- ❌ NEVER use "time pressure" or "user seems rushed" as justification for shortcuts

### **1. LIBRARY DOCUMENTATION FIRST**
**BEFORE implementing ANY custom solution:**
- ✅ Check existing library documentation
- ✅ Research if the tool already provides the needed functionality
- ✅ Use library-native solutions over manual implementations
- ❌ NEVER assume custom implementation is needed without research

### **2. REAL-TIME DOCUMENTATION MANDATE**
**IMMEDIATELY after any learning or problem-solving:**
- ✅ Update relevant .md files with lessons learned
- ✅ Document both what worked AND what didn't work
- ✅ Include root cause analysis for major issues
- ❌ NEVER batch documentation updates - do them instantly

### **3. CODE REMOVAL CONFIRMATION**
**BEFORE removing ANY code or functionality:**
- ✅ Ask user for explicit confirmation
- ✅ Understand the roadmap and future plans
- ✅ Preserve functionality when in doubt
- ❌ NEVER assume something is "unnecessary" without confirmation

### **4. DEEP ANALYSIS MANDATE**
**For EVERY problem or task:**
- ✅ Identify and address root causes, not symptoms
- ✅ Consider system-wide impacts and side effects
- ✅ Apply comprehensive analysis from multiple angles
- ❌ NEVER apply superficial fixes or band-aid solutions

### **5. TYPE SAFETY COMPLIANCE**
**After ANY code modification:**
- ✅ Update TypeScript type definitions immediately
- ✅ Run `npm run type-check` to verify correctness
- ✅ Ensure DRY principle compliance (centralize repeated values)
- ❌ NEVER leave TypeScript errors unresolved

---

## 🤔 **MANDATORY SELF-ASSESSMENT BEFORE ANY IMPLEMENTATION**

Before proceeding with ANY task, answer these questions:
- [ ] **What library am I using?** Have I read its documentation for this feature?
- [ ] **What assumptions am I making?** How can I verify them?
- [ ] **What's the root cause I'm solving,** not just the symptom?
- [ ] **What other parts of the system** might this affect?
- [ ] **How will I know** this solution actually works?
- [ ] **What will I document** and where will I document it?

**If you cannot answer all 6 questions confidently, STOP and research first.**

These mandates override ALL other considerations. If unsure about any directive, STOP and ask for clarification before proceeding.
